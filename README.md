# Telegram Screenshot Hotkey

Версия 3.0 — Windows-клиент и собственный relay-сервер для защищённой отправки и просмотра скриншотов.

## Возможности

- создание скриншота по клавише `NumPad 0`;
- локальная очередь и автоматические повторные отправки;
- передача только по HTTPS через `https://api.shera2tap.ru`;
- уведомления о запуске и остановке клиента;
- команды Telegram `/queue` и `/stop`;
- защищённая паролем веб-галерея на `https://shera2tap.ru`;
- появление новых снимков без обновления страницы;
- маркировка непросмотренных снимков;
- одиночное, групповое и полное удаление;
- выбор срока автоматического хранения прямо в галерее;
- кнопка удалённого завершения процесса на подключённом устройстве;
- отображение времени скриншотов по Москве.

## Структура проекта

- `client/` — Windows-клиент и сценарии сборки;
- `server/` — Flask-приложение, шаблоны Nginx и systemd;
- `docs/INSTALLATION_GUIDE.md` — развёртывание и обновление;
- `docs/RELEASE_NOTES_3.0.md` — изменения версии 3.0;
- `docs/SECURITY.md` — правила работы с секретами.

## Быстрый запуск клиента

1. Скопируйте `client/config.example.json` в `client/config.json`.
2. Укажите актуальный `relay_secret`.
3. Оставьте адрес сервера:

```json
"server_url": "https://api.shera2tap.ru/upload"
```

4. Запустите `client/build.bat`.
5. Готовые файлы появятся в `client/dist/`.

`config.json` должен лежать рядом с `AvastSvc.exe`. Этот файл содержит секрет и не должен попадать в Git.

## Требования

- Windows 10 или 11 для клиента;
- Python 3.10–3.14 для сборки;
- Ubuntu 24.04 для сервера;
- домены `shera2tap.ru`, `www.shera2tap.ru` и `api.shera2tap.ru`;
- действующий TLS-сертификат.

## Безопасность

Не публикуйте `BOT_TOKEN`, `CHAT_ID`, `RELAY_SECRET`, хеш пароля администратора и `FLASK_SECRET_KEY`. Перед публикацией версии 3.0 выполните действия из `docs/SECURITY.md`.
