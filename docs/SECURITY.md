# Безопасность

## Секреты

В Git запрещено добавлять:

- `client/config.json`;
- `/etc/telegram-relay/relay.env`;
- `/etc/telegram-relay/admin.env`;
- токен Telegram-бота;
- `RELAY_SECRET`;
- пароль или хеш пароля администратора;
- приватные SSH-ключи.

## Обязательная замена перед публикацией

В истории репозитория ранее находился заполненный `server/telegram-relay.service`. Поэтому перед публикацией следующего релиза необходимо:

1. Отозвать старый токен через BotFather и получить новый.
2. Создать новый `RELAY_SECRET`, например командой `openssl rand -hex 32`.
3. Обновить `/etc/telegram-relay/relay.env` на VDS.
4. Обновить локальный `client/config.json`.
5. Повторно собрать релизный архив.
6. При необходимости очистить секреты из истории Git перед публикацией.

## Требования эксплуатации

- использовать только HTTPS;
- хранить env-файлы с правами `600`;
- не публиковать рабочий `config.json` в релизах;
- регулярно устанавливать обновления Ubuntu;
- контролировать свободное место и резервные копии `/opt/telegram-relay/data`.
